Prework Study Guide

HTML

  • The head element contains information about the webpage. (not shown on actual page)
  • The body element represents the visible content shown to the user.
  • The section element defines a section that sontains thematic content.
  • Tags: ul-unordered list, ol-ordered list, li-list elements, p-paragraph, br-line break.
  • Headers are opened by h1, h2, etc. Order matters.
  • Footer usually contains copyright and other signatures.

CSS

  • Inline CSS: directly in the HTML file.
  • Internal style sheet: head section of HTML file.
  • External CSS style sheet: all CSS rules in separate file--make design changes easier.
  • Link file using link rel= and href within HTML head.
  • margin: space around the outside of an element.
  • padding: space around the content inside an element.
  • CSS box model: margin-border-padding-content.
  • wild card symbol *: declares that ALL elements will have same style properties.
  • Do Not Repeat Yourself (DRY) code: two or more elements grouped together with the same styling.
  • rel refers to relationship between current doc and linked doc.
  • href specifies URL of external resource (eg. ./assets/style.css --css file in assets folder)
  • . designates class selector
  • color can be assigned by semantic references (gray, blue) or by hexadecimal code or hex #

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it (use '-' to separate names in place of spaces)
  • git stash: discard changes
  • git --help: different git commands you can use
  • git checkout branch-name: move to named branch
  • git add -A: commit all files in branch
  • git commit -m: commit changes with message
  • git pull origin main: pull main branch into feature branch before pushing code

JavaScript

  • Variable: stores data in the code
  • = assigns specific data
  • string contained within quotation marks
  • console.log outputs a message to the web console
  • Chrome DevTools web console: right click, inspect OR command+option+I
  • control flow: order that a computer executes code--JS runs top to bottom
  • conditional statements and loops controls order of execution
  • if (loop): will only execute if code is true
  • else if: allows for more than one condition
  • falsy: 0, -0, On, "", null, undefined, NaN, Boolean data type with value of false
  • truthy: all other values including Boolean value of true
  • === checks if values are equal
  • array: single variable holds group of data--use []
  • for(loop): perform task on each piece of data with few lines of code
  • index: location of data item in array, starts counting at 0
  • starting point (var); condition (var.length); iterate over each item (x++)
  • function: set of instructions of how to perform a task
  • function functionName() { insert for loop}
  • Math.random() generates a random topic from array
